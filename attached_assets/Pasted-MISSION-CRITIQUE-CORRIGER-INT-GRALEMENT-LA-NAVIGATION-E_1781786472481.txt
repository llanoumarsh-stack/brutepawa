══════════════════════════════
BUG 1 : NOTIFICATION → MAUVAISE REDIRECTION
══════════════════════════════

PROBLÈME ACTUEL :

Lorsqu'un utilisateur clique sur une notification :

- L'application s'ouvre.
- Mais la conversation concernée ne s'ouvre pas.
- Ou l'utilisateur est redirigé vers une mauvaise page.

COMPORTEMENT ATTENDU :

Notification message privé :
→ Ouvrir directement la conversation concernée.

Notification groupe :
→ Ouvrir directement le groupe concerné.

Notification commentaire :
→ Ouvrir directement la publication concernée.

Notification demande d'ami :
→ Ouvrir directement le profil concerné.

Notification appel :
→ Ouvrir directement l'écran d'appel.

Notification réaction :
→ Ouvrir directement la publication ou le commentaire concerné.

══════════════════════════════
DEEP LINKING OBLIGATOIRE
══════════════════════════════

Chaque notification doit contenir :

- type
- target_id
- conversation_id
- group_id
- post_id
- user_id

Selon le contexte.

Lors du clic :

Le système doit analyser ces données et naviguer automatiquement vers la page exacte.

Aucun passage par l'accueil.

- Une conversation
- Un groupe

- Une publication

COMPORTEMENT ATTENDU

- La route actuelle
- Les paramètres

- L'identifiant du groupe

Après un refresh :

L'utilisateur doit revenir exactement au même endroit.
