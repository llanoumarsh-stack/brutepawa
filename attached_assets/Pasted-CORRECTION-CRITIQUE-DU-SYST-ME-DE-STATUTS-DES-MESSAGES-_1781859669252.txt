CORRECTION CRITIQUE DU SYSTÈME DE STATUTS DES MESSAGES

BUG ACTUEL :
Dans la liste des conversations, les icônes ✓ et ✓✓ apparaissent correctement puis disparaissent après quelques secondes ou changent d'état de manière incohérente.

OBJECTIF :
Reproduire exactement le comportement de Telegram, WhatsApp et Messenger.

RÈGLES OFFICIELLES :

✓ gris = message envoyé au serveur

✓✓ gris = message reçu par le destinataire

✓✓ vert = message lu par le destinataire

Une fois qu'un message atteint un état supérieur, il ne doit JAMAIS revenir à un état inférieur.

INTERDICTIONS :

read → delivered ❌

read → sent ❌

delivered → sent ❌

read_at

Valeurs autorisées :

sent
delivered

Lorsqu'un message est envoyé :

status = sent

Lorsqu'il arrive sur l'appareil du destinataire :

status = delivered

Enregistrer delivered_at

Lorsqu'il est ouvert ou vu :

status = read

Enregistrer read_at

SYNC TEMPS RÉEL :

Supabase Realtime doit envoyer immédiatement :

message_delivered
message_read

Tous les clients doivent être mis à jour instantanément.

PROTECTION ANTI-RÉGRESSION :

const priority = {
 sent: 1,
 delivered: 2,
 read: 3
};

if (
 priority[newStatus] >
 priority[currentStatus]
) {
 updateStatus(newStatus);
}

Ne jamais remplacer un statut plus avancé par un statut plus ancien.
